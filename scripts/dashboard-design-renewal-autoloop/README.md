# Dashboard Design Renewal Autoloop

Codex-driven autonomous loop for completing the common dashboard design renewal described in `docs/design-dashboard-shell.md`.

## Start

```bash
bash scripts/dashboard-design-renewal-autoloop/run-loop.sh
```

The default verification gate is `foundation`. A `DONE` footer from Codex is accepted only after the full dashboard renewal gate passes.

## Useful Options

```bash
MAX_ITERATIONS=20 bash scripts/dashboard-design-renewal-autoloop/run-loop.sh
AUTOCOMMIT=true bash scripts/dashboard-design-renewal-autoloop/run-loop.sh
CODEX_ARGS="--full-auto" bash scripts/dashboard-design-renewal-autoloop/run-loop.sh
VERIFY_STAGE=react-build bash scripts/dashboard-design-renewal-autoloop/run-loop.sh
DONE_VERIFY_STAGE=full bash scripts/dashboard-design-renewal-autoloop/run-loop.sh
VERIFY_STAGE=full bash scripts/dashboard-design-renewal-autoloop/verify.sh
```

## Verification Stages

- `foundation`: shell syntax, Go tests, and dashboard service registry tests.
- `react-build`: installs existing dependencies if needed and runs the React dashboard build.
- `api-smoke`: starts a temporary `devcloud` instance and checks `/`, `/mail`, `/s3`, and `/api/dashboard/services`.
- `full`: runs all gates above.

## Files

- `goal.md`: measurable implementation contract.
- `bootstrap.sh`: initializes resumable state and validates shell syntax.
- `run-loop.sh`: Codex runner with bounded iterations, lock, status parsing, verification gates, circuit breaker, and checkpoint writes.
- `verify.sh`: acceptance verification gate.
- `recover.sh`: rebuilds `state.env` from `progress.md` and clears stale lock/circuit state.
- `progress.md`: append-only loop timeline generated by the runner.
- `state.env`: resumable execution state generated by the runner.

The runner expects Codex CLI to be available as `codex`.
