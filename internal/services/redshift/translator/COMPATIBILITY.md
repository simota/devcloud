# Redshift ↔ PostgreSQL Compatibility Checklist

devcloud の Redshift サービスは、`translator.RedshiftToPostgres` で Redshift SQL を PostgreSQL に書き換えてから managed PostgreSQL backend に渡します。本ファイルは「Redshift と PostgreSQL の構文差分」と「translator の対応状況」を網羅する **チェックリスト** です。新しい翻訳ルールを追加する際の優先度判断と進捗管理に使います。

## 凡例

- **R-only**: Redshift にしか存在しない。devcloud は素通しで PostgreSQL に渡すため `syntax error` / `undefined function` になる。
- **R≠P**: 名前は同じ／似ているが、引数・挙動・型・戻り値・精度が異なる（**翻訳が必要なのに見過ごされやすい罠**）。
- **P-only**: PostgreSQL にしかない。devcloud では「動いてしまう」が本物の Redshift では失敗する **偽陽性**。
- `- [x]` = translator で対応済み（または別経路で実装済み） / `- [ ]` = 未対応 / 要検討

---

## 1. データ型

- [x] **R-only** `SUPER` — semi-structured。PG `jsonb` にマップするのが現実的
- [x] **R-only** `HLLSKETCH` — HyperLogLog スケッチ
- [x] **R-only** `GEOMETRY` / `GEOGRAPHY` — Redshift 固有実装（PG は PostGIS 拡張別）
- [x] **R-only** `VARBYTE` — PG `bytea` にマップ
- [x] **R≠P** `TIMESTAMP` — μs 精度、TZ デフォルトの扱いが微妙に異なる
- [x] **R≠P** `TIMESTAMPTZ` — Redshift は UTC 固定保存、PG は session TZ
- [x] **R≠P** `TIME` / `TIMETZ` — Redshift は集約・サブクエリで一部使用不可
- [x] **R≠P** `BOOLEAN` リテラル — Redshift は `0/1/t/f/y/n/yes/no/true/false` 全部、PG は厳密
- [x] **R≠P** `CHAR(n)` / `VARCHAR(n)` — Redshift は **バイト数**、PG は **文字数**
- [x] **R≠P** `IDENTITY(seed, step)` → `GENERATED BY DEFAULT AS IDENTITY` （`translateCreateTable` で対応）
- [x] **same** `SMALLINT` / `INTEGER` / `BIGINT` / `REAL` / `DOUBLE PRECISION` / `NUMERIC`
- [ ] **P-only** `JSONB` の演算子・インデックス、`int[]` 配列、`RANGE`、`UUID`、`BIT(n)` — Redshift にない（devcloud では通ってしまう）

---

## 2. DDL — CREATE TABLE / ALTER TABLE

- [x] **R-only** `DISTSTYLE {EVEN|KEY|ALL|AUTO}` — 剥がしてカタログ保存
- [x] **R-only** `DISTKEY (col)` / 列レベル `col TYPE DISTKEY`
- [x] **R-only** `SORTKEY [COMPOUND|INTERLEAVED] (cols)` / 列レベル `SORTKEY` ※`INTERLEAVED` は記録のみで挙動再現なし
- [x] **R-only** `ENCODE <type>` (LZO/ZSTD/AZ64/RAW/DELTA/BYTEDICT/MOSTLY8 等)
- [x] **R-only** `BACKUP {YES|NO}`
- [x] **R-only** `CREATE EXTERNAL TABLE ... LOCATION 's3://...'` (Spectrum)
- [x] **R-only** `CREATE EXTERNAL SCHEMA FROM DATA CATALOG`
- [x] **R-only** `CREATE MATERIALIZED VIEW ... AUTO REFRESH YES`
- [x] **R-only** `WITH NO SCHEMA BINDING` (late-binding view)
- [x] **R-only** `ALTER TABLE ... ALTER COLUMN ... ENCODE ...`
- [x] **R-only** `ALTER TABLE ... ADD COLUMN ... DEFAULT IDENTITY (...)`
- [x] **R≠P** `CREATE TABLE ... LIKE` のオプション (`INCLUDING DEFAULTS` 等) の差異
- [x] **R≠P** `TEMP` / `TEMPORARY` テーブルのスコープ規則
- [ ] **P-only** `CREATE INDEX` — Redshift に索引なし、devcloud は通ってしまう
- [ ] **P-only** `CREATE EXTENSION` — Redshift に概念なし
- [ ] **P-only** `INHERITS (...)` テーブル継承
- [ ] **P-only** `PARTITION BY RANGE/LIST/HASH` — Redshift は外部表のみ

---

## 3. DML

- [x] **R-only** `MERGE INTO ... WHEN MATCHED THEN UPDATE ... WHEN NOT MATCHED THEN INSERT` — PG 15+ で同名構文あり、それ未満は要書き換え
- [x] **R-only** `COPY <table> FROM 's3://...' IAM_ROLE ...` — 独自実装（`sql_copy_unload.go`）
- [x] **R-only** `UNLOAD ('SELECT...') TO 's3://...'` — 独自実装
- [x] **R-only** `TRUNCATE` の即時コミット仕様（Redshift はトランザクション内でロールバック不可）
- [x] **R≠P** `INSERT INTO ... VALUES (DEFAULT)` の identity 列挙動
- [x] **R≠P** `INSERT INTO ... SELECT` の `RETURNING` 句（Redshift にはない、PG にはある）
- [ ] **P-only** `INSERT ... ON CONFLICT DO UPDATE` (UPSERT)
- [ ] **P-only** `WITH ... DELETE/UPDATE` (writable CTE)
- [ ] **P-only** `RETURNING` 句

---

## 4. SELECT / クエリ構文

- [x] **R-only** `QUALIFY` 句 — サブクエリで代用
- [x] **R-only** `SELECT TOP n ...` — PG `LIMIT n` に書き換え
- [x] **R-only** `LIKE ... ESCAPE` の `\` デフォルトエスケープ挙動
- [x] **R-only** `APPROXIMATE COUNT(DISTINCT col)` — 近似不要なら `COUNT(DISTINCT)` に置換可
- [x] **R-only** `LISTAGG(expr, sep) WITHIN GROUP (ORDER BY ...)` → `string_agg(expr, sep ORDER BY ...)`
- [x] **R-only** `MEDIAN(col)` → `percentile_cont(0.5) WITHIN GROUP (ORDER BY col)`
- [x] **R-only** `RATIO_TO_REPORT(col) OVER (...)` → `col / SUM(col) OVER (...)`
- [x] **R-only** `LATERAL` の制限 — PG はフル LATERAL
- [x] **R≠P** `NULLS FIRST/LAST` 既定の差（実際は同じだが要確認）
- [x] **R≠P** `BIT_AND/BIT_OR(boolean)` 集約 → PG `bool_and/bool_or`
- [x] **R≠P** `LISTAGG` の Window 形 — PG `string_agg` は Window 非対応、要 workaround
- [ ] **P-only** `FETCH FIRST n ROWS WITH TIES`
- [ ] **P-only** `TABLESAMPLE`
- [ ] **P-only** Window 関数の `EXCLUDE GROUP/TIES/CURRENT ROW` 句

---

## 5. スカラー関数 — 日付・時刻

- [x] **R-only** `GETDATE()` → `CURRENT_TIMESTAMP`
- [x] **R-only** `SYSDATE` → `CURRENT_TIMESTAMP`
- [x] **R-only** `DATEADD(part, n, ts)` → `ts + n * interval '1 part'`
- [x] **R-only** `DATEDIFF(part, a, b)` → `date_part` / `age` / `extract(epoch)` の組み合わせ
- [x] **R≠P** `DATE_PART(part, ts)` / `DATE_TRUNC(part, ts)` の `part` 許容名（`weekday`/`dow` 等のマッピング拡張）
- [x] **R-only** `CONVERT_TIMEZONE('UTC', 'JST', ts)` → `ts AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Tokyo'`
- [x] **R-only** `TIMEOFDAY()` — PG `clock_timestamp()` を文字列化
- [x] **R-only** `LAST_DAY(date)` → `(date_trunc('month', date) + interval '1 month - 1 day')::date`
- [x] **R-only** `MONTHS_BETWEEN(a, b)` → `extract(year from age(a, b)) * 12 + extract(month from age(a, b))`
- [x] **R-only** `NEXT_DAY(date, dow_string)`
- [x] **R-only** `ADD_MONTHS(date, n)` → `date + n * interval '1 month'`
- [ ] **R≠P** `TO_DATE` / `TO_TIMESTAMP` の書式コード差（`TZ` 等）
- [ ] **R≠P** `TO_CHAR(ts, fmt)` の書式コード差

---

## 6. スカラー関数 — 文字列

- [ ] **R-only** `LEN(s)` → `length(s)` または `char_length(s)`
- [ ] **R-only** `CHARINDEX(substring, string)` → `position(substring in string)`
- [ ] **R-only** `STRTOL(text, base)` — 代用式
- [ ] **R-only** `CRC32(text)` — PG に同等なし
- [ ] **R-only** `FUNC_SHA1` / `MD5_DIGEST` — PG は `md5()` のみ、SHA は拡張
- [ ] **R-only** `REGEXP_SUBSTR(s, pattern, start, occurrence)` → `regexp_match(s, pattern)` 配列返し
- [ ] **R-only** `REGEXP_COUNT(s, pattern)`
- [ ] **R-only** `REGEXP_INSTR(s, pattern, ...)`
- [ ] **R≠P** `SPLIT_PART(s, sep, n)` 負数 n の挙動（PG 14+ で対応）
- [ ] **R≠P** `SUBSTRING(s, start, len)` 負の `start` の扱い

---

## 7. スカラー関数 — 数値・条件

- [x] **R-only** `NVL(a, b)` → `COALESCE(a, b)`
- [ ] **R-only** `NVL2(expr, val_if_not_null, val_if_null)` → `CASE WHEN expr IS NOT NULL THEN ... ELSE ... END`
- [x] **R-only** `DECODE(expr, k1, v1, k2, v2, default)` → `CASE`
- [ ] **R-only** `RAND()` 大文字 → PG `random()`
- [ ] **R≠P** `GREATEST` / `LEAST` の NULL 扱い（Redshift は NULL 無視、PG は NULL 返す）
- [ ] **R≠P** `ROUND(n, d)` 負の `d` 挙動の微差

---

## 8. JSON / SUPER

- [ ] **R≠P** `JSON_EXTRACT_PATH_TEXT(json, p1, p2, ..., [null_if_invalid])` — 第 N+1 引数 boolean の有無で型不一致
- [ ] **R-only** `JSON_EXTRACT_ARRAY_ELEMENT_TEXT(json, idx)` → `(json::jsonb -> idx)::text`
- [ ] **R-only** `JSON_ARRAY_LENGTH(json)` → `jsonb_array_length(json::jsonb)`
- [ ] **R-only** `JSON_PARSE(text)` → `text::jsonb`
- [ ] **R-only** `OBJECT_TRANSFORM(super_value, ...)` — PG に同等なし
- [ ] **R-only** `IS_VALID_JSON` / `IS_VALID_JSON_ARRAY` — PG 16+ `json_valid()`
- [ ] **R-only** PartiQL ドット記法 (`super_col.key.subkey`) / 添字 (`super_col[0]`) → PG `->`, `->>`

---

## 9. システムカタログ / メタデータ

- [ ] **R-only** `STV_*` (運用統計、リアルタイム) — read-only スタブの実装余地あり
- [ ] **R-only** `STL_*` (ログ、過去)
- [ ] **R-only** `SVV_*` / `SVL_*` / `SYS_*` ビュー
- [ ] **R-only** `PG_TABLE_DEF` / `PG_TABLE_INFO` (Redshift 拡張カラム)
- [x] **same** `pg_catalog.pg_class` / `pg_namespace` / `pg_attribute`
- [ ] **R≠P** `information_schema.*` の列差異

---

## 10. トランザクション / セッション

- [ ] **R-only** `BEGIN [READ ONLY|READ WRITE] [ISOLATION LEVEL SERIALIZABLE]` — Redshift は SERIALIZABLE のみ
- [x] **same** `SET search_path` の有効範囲
- [ ] **R≠P** `RESET` コマンドの一部挙動差
- [ ] **P-only** `SAVEPOINT`
- [ ] **P-only** `LISTEN` / `NOTIFY`
- [ ] **P-only** `PREPARE` / `EXECUTE` / `DEALLOCATE` の拡張機能

---

## 11. 権限 / セキュリティ

- [ ] **R-only** `GRANT ASSUMEROLE ON 'arn:...' TO ...`
- [ ] **R-only** `CREATE DATASHARE` / `ALTER DATASHARE` / `GRANT USAGE ON DATASHARE`
- [ ] **R-only** `CREATE MASKING POLICY` / `ATTACH MASKING POLICY`
- [ ] **R-only** `CREATE ROW ACCESS POLICY` — PG は RLS、構文差
- [ ] **R≠P** `CREATE USER ... PASSWORD DISABLE` 等のパスワード句
- [ ] **P-only** `CREATE ROLE WITH LOGIN BYPASSRLS`

---

## 12. プロシージャ / UDF / ML

- [ ] **R-only** `CREATE OR REPLACE PROCEDURE ... LANGUAGE plpgsql` の OUT/INOUT 制限差
- [ ] **R-only** `CREATE FUNCTION ... LANGUAGE plpythonu` (Python UDF)
- [ ] **R-only** `CREATE FUNCTION ... LANGUAGE sql STABLE` の制約差
- [ ] **R-only** `CREATE MODEL ... FROM ... TARGET ... FUNCTION ... IAM_ROLE ...` (Redshift ML)
- [ ] **R-only** `CREATE EXTERNAL FUNCTION ... LAMBDA 'arn:...'`
- [ ] **R≠P** `EXPLAIN` の Redshift 専用注釈

---

## 13. データロード / アンロード / 連携

- [x] **R-only** `COPY <table> FROM 's3://...' IAM_ROLE ... [JSON 'auto'] [PARQUET]` — `sql_copy_unload.go`
- [x] **R-only** `UNLOAD ('SELECT...') TO 's3://...' PARALLEL OFF ALLOWOVERWRITE` — `sql_copy_unload.go`
- [ ] **R-only** `CREATE EXTERNAL FUNCTION ... LAMBDA 'arn:...'`

---

## 14. R≠P 同名関数の罠（要注意：翻訳しないと意味的誤動作）

- [ ] `JSON_EXTRACT_PATH_TEXT(json, k1, k2, [null_if_invalid])` — Redshift は最後の引数が boolean、PG は variadic text のみ
- [ ] `LEN(s)` (Redshift) vs `LENGTH(s)` (PG) — 単純な名前差
- [ ] `CHAR(n)` / `VARCHAR(n)` のバイト数 vs 文字数 — 多バイト文字でカラム長エラーが本番でだけ出る
- [ ] `EXTRACT(week FROM ts)` — 両方 ISO week なので OK だが要文書化
- [ ] `TO_CHAR(ts, 'TZ')` — Redshift と PG で書式コードに差
- [ ] `LISTAGG ... OVER (...)` の Window 形 — PG では `string_agg` が Window 非対応
- [ ] `GREATEST` / `LEAST` の NULL 扱い差

---

## 15. P-only（devcloud で「動いてしまう」偽陽性）

devcloud で書いた SQL がローカルでは通っても、本物の Redshift では失敗する系。translator の前段で **禁止オプション** として弾く拡張余地あり（オプトイン）。

- [ ] `INSERT ... ON CONFLICT DO UPDATE` (UPSERT)
- [ ] `RETURNING` 句
- [ ] `CREATE INDEX`
- [ ] `CREATE EXTENSION`
- [ ] `INHERITS (...)` テーブル継承
- [ ] `PARTITION BY RANGE/LIST/HASH`
- [ ] `FETCH FIRST n ROWS WITH TIES`
- [ ] `TABLESAMPLE`
- [ ] Window `EXCLUDE` 句
- [ ] `SAVEPOINT`
- [ ] `LISTEN` / `NOTIFY`
- [ ] `JSONB` 演算子 (`@>`, `?`, `#>>`, etc.)
- [ ] 配列型・`UUID`・`RANGE` 型

---

## 既存対応サマリ

translator で対応済み (`internal/services/redshift/translator/translator.go` 参照):

- スカラー関数: `GETDATE` / `SYSDATE` / `NVL` / `DECODE` / `DATEADD` / `DATEDIFF` / `LISTAGG`
- CREATE TABLE 属性: `DISTKEY` / `SORTKEY` / `DISTSTYLE` / `ENCODE` / `BACKUP` / `IDENTITY`

別経路で実装済み:

- `COPY` / `UNLOAD` (`internal/services/redshift/sql_copy_unload.go`)

---

## 推奨追加優先度

1. **使用頻度が高く翻訳容易**: `LEN`, `CHARINDEX`, `NVL2`, `CONVERT_TIMEZONE`, `MEDIAN`, `LAST_DAY`, `ADD_MONTHS`, `MONTHS_BETWEEN`, `RAND`
2. **データ型ブリッジ**: `SUPER` → `jsonb`、`VARBYTE` → `bytea`、PartiQL ドット記法 → `->>`
3. **頻出 R-only 構文**: `QUALIFY`, `SELECT TOP n`, `APPROXIMATE COUNT(DISTINCT)`, `MERGE INTO` (PG 15+ チェック付き)
4. **R≠P 罠の検出と書き換え**: `JSON_EXTRACT_PATH_TEXT` の引数差、`GREATEST`/`LEAST` の NULL 扱い
5. **システム表の read-only スタブ**: `STV_TBL_PERM`, `SVV_TABLE_INFO` などダッシュボード・運用ツール対応の最小集合
6. **P-only 偽陽性検出**: translator の前段で禁止オプションとして警告/拒否（オプトイン）
